## Scope
### Must have
- `packages/core/src/protocol/`: a `ProtocolAdapter` contract (interface + per-protocol adapter objects, composition NOT class inheritance) wrapping the existing ingress/transform/egress functions for all 4 protocols: openai-completions, openai-responses, anthropic, gemini.
- ONE generic route pipeline `createProtocolRoutes(adapter, source, ...)` in `packages/server` replacing the 4 hand-rolled pipelines in `packages/server/src/routes/*.ts`; route files become thin registrations keeping only URL quirks (gemini model-in-path + `:streamGenerateContent`, anthropic `/count_tokens`, responses `GET /v1/responses/:id` 501).
- `Router.resolve(model)` returns an ORDERED candidate list (config order). Cross-provider duplicate alias = fallback order (no longer `RouterModelCollisionError`); same-provider duplicate alias still errors. Pipeline tries candidates in order per AGENTS.md; final failure preserved.
- API->AI-SDK bridge: protocol-mismatched `api` provider invoked through the matching AI SDK package built from its protocol/baseUrl/apiKey metadata (`openai-completions`->`@ai-sdk/openai-compatible`, `openai-responses`->`@ai-sdk/openai`, `anthropic`->`@ai-sdk/anthropic`, `gemini`->`@ai-sdk/google`). Zero new dependencies (all in workspace catalog, `packages/core/src/provider/ai-sdk-loader.ts:5-14`).
- Naming unification (USER DIRECTIVE): every internal `openai-chat` expression becomes `openai-completions` (files, exported symbols, error tags, fixtures dir). `ProviderProtocol` enum: `OpenAICompatible = "openai-compatible"` -> `OpenAICompletions = "openai-completions"`; `OpenAIResponse = "openai-response"` -> `OpenAIResponses = "openai-responses"`. NO back-compat aliases (project unreleased, user-confirmed).
- Drift-bug fixes folded in: gemini route gains the 8MiB body guard; openai-responses errors become responses-shaped (no longer forced through chat-only `toIngressError`); anthropic non-stream egress moves to core with tool_use blocks + usage + correct stop_reason (route-local lossy `aiSdkMessages`/`anthropicMessage` deleted).
- Provider probes sourced from adapters (`providerProbeRequest` switch in `packages/server/src/provider-runtime.ts:134-169` deleted).
- AGENTS.md updated: adapter architecture, new enum vocabulary, fallback semantics.

### Must NOT have (guardrails, anti-slop, scope boundaries)
- NO new npm dependencies.
- NO weighted routing, retry policies, circuit breakers, or health-based candidate skipping (config order IS the weight, per AGENTS.md).
- NO subscription-provider invocation (skipped in candidate loop; if the only candidate, respond 501 unsupported).
- NO dashboard/CLI feature work beyond what compilation requires.
- NO change to same-protocol raw passthrough mechanics (byte-level proxy in `packages/core/src/provider/api.ts:53-79` untouched: header rewrite, tee-trace, encoding normalization stay as-is).
- NO abstract class hierarchy; the contract is a type + plain adapter objects (repo is function-first; classes only for Errors).
- NO renaming of HTTP endpoint paths, config field NAMES, or the `packageName` default `@ai-sdk/openai-compatible` (that is an npm package name, not the protocol vocabulary).
- NO speculative extension points (no 5th-protocol scaffolding, no plugin registry).
- Do NOT delete `RouterModelCollisionError` or the `error_alias_collision` i18n keys - same-provider duplicates still use them.
